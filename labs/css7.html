CSS, Part 2

IDs

There are times when you want to define the properties of an entire area or division of your page, rather than setting the page's properties one tag at a time. You can do this by defining an ID in your CSS file:

#content {
    font-family: verdana,arial,sans-serif;
    font-size: 1.1em;
    line-height: 1.6em;
    color: #333;
}

Note that while a class name in your CSS begins with a period, an ID name begins with a pound sign. The difference between classes and IDs is an important one: classes can be applied repeatedly within your HTML file, and are used to define elements on the page that you want styled similarly. IDs, by contrast, can only be used once on any given page, and are usually used to define a region of a page (like a navigation bar) that is used on many different pages.

We would use our "content" ID by including code like this in our page:

<div id="content">This is the page's content.</div>

When you actually include this code in your HTML document, it winds up looking like this:

This paragraph should be rendered in a sans-serif font (preferably Verdana), it should be slightly larger than average, with reasonable line spacing, and it should be dark gray in color.
