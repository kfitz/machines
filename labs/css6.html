CSS, Part 2

Classes

Initially, when we started working with CSS, we focused on changing the ways that a browser renders standard HTML tags, like <h1>, <p>, and the like, with examples such as this:

h1 {
    color: #990066;
    font-weight: bold;
    font-size: 40px;
    font-family: helvetica,arial,sans-serif;
    text-align: center;
}
p {
    margin-top: 6px;
    margin-left: 70px;
    margin-right: 70px;
    color: #000000;
}

However, it's also possible to define multiple ways for a single HTML tag to render, by using classes. For instance, you could define a class called "blue," by including something like this in your stylesheet:

.blue {
    color: #0000CC;
}

This code creates a class called "blue" that can be applied to any tag used in your HTML document, like this:

<p class="blue">This paragraph should render as blue.</p>

Which actually winds up looking like this:

This paragraph should be blue.

You can use a class like this with any other appropriate tag -- for instance, an h2:

This subhead should be blue.

Everything else about the formatting is kept from the general stylesheet instructions for the tag; only the properties specified by the class are overridden. (Be careful, however; if you write conflicting properties into different parts of your CSS, you might get unpredictable results. It's best to define a property such as font-family or color in one and only one tag.)
